    How It Works

    1. UI Starts A Session

    The button calls the UI backend, which creates a new game session through the session service.

    2. Session Service Simulates

    The session service generates alternating X and O moves and sends them to the game engine over REST.

    3. Engine Validates

    The game engine validates moves, updates board state, and reports whether the game is still running, won, or drawn.
